# TaskFlow — Claude Code ハンズオン

シンプルなカンバン形式のタスクマネージャーです。**意図的なバグ**と**未実装の機能**が含まれており、約1時間のClaude Codeワークショップ用に設計されています。

## クイックスタート

```bash
# ブラウザで開く（ビルド不要）
open index.html
```

## 演習1 — バグ探し（15分）

`app.js` に**6つのバグ**が隠されています。Claude Codeを使って見つけて修正してください。

**プロンプトのヒント：**
- 「app.js にあるバグをすべて見つけて、それぞれ説明してください」
- 「app.js のバグ#3を修正してください」
- 「タスク数が '1 tasks' と表示されている — 文法を修正してください」

### バグ一覧（ネタバレ注意 — まず自分で探してみましょう！）

<details>
<summary>バグを表示する</summary>

| # | 場所 | バグ |
|---|------|------|
| 1 | `formatDueDate()` | 日付がUTC深夜としてパースされ、タイムゾーンのずれが発生する |
| 2 | `getFilteredTasks()` | 検索が `title` のみをチェックし、`description` や `assignee` をチェックしない |
| 3 | `render()` | カラムのカウントが、そのステータスの合計数ではなくフィルター後の件数を表示する |
| 4 | `render()` | 「X tasks」が「1 task」の単数形を使用しない |
| 5 | `deleteTask()` | 確認ダイアログがない — タスクが即座に削除され、元に戻せない |
| 6 | `saveToStorage()` | `nextId` が保存されない — 更新後、新しいタスクが重複したIDを取得する |

</details>

## 演習2 — 新機能追加（20分）

以下の機能のうち**1つ**を追加してください：

### オプションA — 優先度ソート
各カラム内のタスクを優先度（高 → 中 → 低）でソートする。

**スタータープロンプト：**
> 「各カラムにソートボタンを追加して、優先度の高いものから順にタスクをソートしてください。」

### オプションB — ダークモード
ヘッダーにダークモード切り替えボタンを追加する。

**スタータープロンプト：**
> 「ヘッダーにダークモード切り替えボタンを追加してください。設定をlocalStorageに保存してください。」

### オプションC — タスクタグ
ユーザーがカンマ区切りのタグをタスクに追加し、タグでフィルタリングできるようにする。

**スタータープロンプト：**
> 「タスクフォームにタグフィールドを追加してください。各カードにタグを小さなバッジとして表示してください。フィルターバーにタグフィルターのドロップダウンを追加してください。」

### オプションD — 期限日ソート
「期限日でソート」オプションを追加し、2日以内に期限が来るタスクをハイライト表示する。

**スタータープロンプト：**
> 「各カラム内のタスクを期限日でソートしてください。2日以内に期限が来るタスクを黄色のボーダーでハイライトしてください。」

## 演習3 — リファクタリング（15分）

`render()` と `renderTask()` 関数が多くの処理を担いすぎています。

**プロンプトのアイデア：**
- 「renderTask() をより読みやすくリファクタリングしてください」
